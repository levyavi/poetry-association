# Task 002: Text Cleaning, Embedding Service, Poem Repository, and CSV Import CLI

## 0. Complexity Assessment
- Level: High
- Justification:
  - Technical complexity: Integrates sentence-transformers model loading, numpy serialization, SQLite persistence, and CSV parsing — four distinct subsystems.
  - Integration complexity: First task with cross-cutting data flow (filesystem → CSV → cleaning → embedding → SQLite BLOB).
  - Risk level: Medium-high — model loading failure, offline model availability, and binary embedding serialization are real failure modes.
  - Estimated effort: ~500 changed lines across modules and tests.
- Complexity rubric alignment:
  - Low: single component, no persistence, minimal side effects
  - Medium: multiple components and/or persistence involved
  - High: cross-cutting concerns, filesystem + DB interaction, concurrency, or irreversible operations ← fits here (filesystem + DB + model IO).
- Recommended execution mode:
  - High → incremental implementation with validation checkpoints (cleaning → repository → embedding → import CLI).

---

## 1. Goal
Implement the core data pipeline — text cleaning, the local embedding service (`all-MiniLM-L6-v2`), the poem repository, and a `import-csv` CLI — so poems can be loaded into SQLite with normalized text and stored embeddings, providing a verifiable data foundation for search.

---

## 2. Why this task exists now
Search, admin CRUD, CSV import UI, and rebuild all depend on a working cleaning + embedding + persistence path. Exercising that pipeline from a CLI first delivers an observable, testable milestone without needing the public search UI or admin authentication, keeping the slice vertical and deployable.

## 10. Classes to Implement or Modify
### EmbeddingService
- Responsibility: Own the loaded sentence-transformers model and expose a small interface for encoding and normalization.
- Key methods: `__init__`, `encode`, `encode_query`, `to_bytes`, `from_bytes`.
- Interactions: Called by `repository.create_poem`, the CLI import, and later by the search service in Task 003.
- Constraints: Loads the model exactly once per process; never connects to the network; raises a clear error if the model is not found locally.

---

## 11. Public Functions / Methods
### text_cleaning.clean_poem_text
- Purpose: Trim, normalize line endings, collapse excessive blank lines, and tidy inline whitespace without lowercasing or removing structural line breaks.
- Parameters: `text: str`.
- Returns: cleaned `str` preserving case and meaningful line breaks.
- Side effects: none.
- Errors: raises `ValueError` on `None` input.

### text_cleaning.compute_dedup_key
- Purpose: Produce the canonical form used for duplicate detection per design doc §6.3.3 (whitespace normalized, case preserved).
- Parameters: `cleaned_text: str`.
- Returns: normalized string.
- Side effects: none.
- Errors: none.

### text_cleaning.clean_query
- Purpose: Apply query normalization — trim, lowercase, normalize punctuation spacing — per design doc §7.2.
- Parameters: `query: str`.
- Returns: normalized query `str` or empty string.
- Side effects: none.
- Errors: none.

### EmbeddingService.encode
- Purpose: Encode a poem (title + body combined per design doc §8.3) into a unit-normalized numpy `float32` array.
- Parameters: `title: str`, `text: str`.
- Returns: `np.ndarray` of shape `(dim,)` with L2 norm of 1.
- Side effects: none.
- Errors: propagates model errors.

### EmbeddingService.encode_query
- Purpose: Encode a cleaned query string into a unit-normalized vector.
- Parameters: `query: str`.
- Returns: `np.ndarray`.
- Side effects: none.
- Errors: propagates model errors.

### EmbeddingService.to_bytes / from_bytes
- Purpose: Serialize and deserialize embeddings for SQLite BLOB storage.
- Parameters: `vector`, or `blob`.
- Returns: `bytes` or `np.ndarray`.
- Side effects: none.

### repository.create_poem
- Purpose: Insert a poem row with cleaned text and embedding. Rejects duplicates by cleaned text.
- Parameters: `conn`, `title: str`, `text: str`, `embedding_service`.
- Returns: inserted poem `id`.
- Side effects: DB write.
- Errors: raises `DuplicatePoemError` if a poem with the same `cleaned_text` already exists; propagates embedding errors.

### repository.get_poem
- Purpose: Fetch a single poem row.
- Parameters: `conn`, `id`.
- Returns: `sqlite3.Row` or `None`.

### repository.list_poems
- Purpose: Return all poems ordered by the provided sort key (default `title ASC`).
- Parameters: `conn`, `order_by`.
- Returns: list of rows.

### repository.find_by_cleaned_text
- Purpose: Look up a poem by its canonical dedup key.
- Parameters: `conn`, `dedup_key`.
- Returns: `sqlite3.Row` or `None`.

### csv_import.plan
- Purpose: Parse the strict CSV format, normalize each row, classify as importable or duplicate (against both the existing DB and prior rows in the same file), return a plan object.
- Parameters: `conn`, `csv_path_or_stream`.
- Returns: `ImportPlan` containing `importable_rows`, `duplicate_count`, `filename`.
- Side effects: none (read-only).
- Errors: `CsvFormatError` on missing or wrong headers, encoding problems, or malformed rows.

### csv_import.execute
- Purpose: Walk an `ImportPlan` row-by-row, insert each poem using `repository.create_poem`, checking an optional `cancel_flag` before each row, returning counts.
- Parameters: `conn`, `plan`, `embedding_service`, `cancel_flag` (optional callable returning bool), `on_progress` (optional callable).
- Returns: `ImportResult` with `imported`, `skipped_duplicates`, `cancelled`, `error`.
- Side effects: DB writes.
- Errors: stops on first row-level error after imports; prior rows remain committed.

### cli.import_csv
- Purpose: CLI entry point wiring config, DB, embedding service, plan, and execute together.
- Parameters: argv.
- Returns: process exit code.
- Side effects: writes to stdout.
- Errors: prints a readable error and exits with non-zero on failure.

## 12. Data Model / Schema Changes
- Database changes: none (schema already created by Task 001).
- File formats:
  - CSV: strict UTF-8, comma-delimited, header row `title,text` required. No other headers, no alternative delimiters.
  - Embedding BLOB: `np.float32` little-endian bytes with leading 4-byte unsigned int for dimension (enables safe deserialization and future-proofing).
- In-memory structures:
  - `ImportPlan` dataclass: list of `PlannedRow(title, text, cleaned_text, dedup_key)`, `duplicate_count`, `filename`.
  - `ImportResult` dataclass: `imported`, `skipped_duplicates`, `cancelled`, `error`.

## 14. State Transitions
- Before:
  - `poems` table exists but is empty (or contains poems from prior imports).
  - Embedding model is available locally but not yet loaded.
- After:
  - Poems referenced in the imported CSV are persisted with cleaned text and embeddings.
  - Duplicates (based on cleaned dedup key) were skipped.
  - Model is loaded once in the running process.
- Invariants:
  - Every row in `poems` has a non-null `embedding` BLOB of the correct dimension.
  - No two rows share the same dedup key.
  - `cleaned_text` preserves case and line breaks per design doc §9.

## 15. Detailed Implementation Notes
1. Implement `text_cleaning.py` first with a small, deterministic rule set. `clean_poem_text`: strip leading/trailing whitespace; normalize `\r\n` and `\r` to `\n`; collapse 3+ blank lines to at most 2; strip trailing whitespace per line; preserve case, indentation, and internal line breaks.
2. `compute_dedup_key` further collapses internal whitespace runs to single spaces and joins lines with `\n`, case preserved — this matches design doc §6.3.3 exactly.
3. `clean_query`: strip, lowercase, collapse whitespace, normalize smart quotes to ASCII, strip redundant punctuation spacing.
4. Implement `EmbeddingService` using `sentence_transformers.SentenceTransformer(model_name_or_path)`; use `model.encode(text, normalize_embeddings=True)` so output is already unit-normalized. Coerce to `np.float32`.
5. Serialization format: pack dimension as uint32 then raw float32 bytes. Deserialization validates dimension matches the loaded model.
6. In `create_app`, after `init_db`, build the embedding service and attach it at `app.extensions["embedding"]`. Print `embedding model ready (<name>)` before the HTTP server starts.
7. `repository.create_poem` flow: clean text, compute dedup key, `find_by_cleaned_text(dedup_key)` → raise `DuplicatePoemError` if found; else encode title+text, serialize, `INSERT` row with timestamps, return id. Use a single transaction.
8. `csv_import.plan` uses `csv.DictReader` with `encoding="utf-8-sig"` to tolerate BOM. Required fieldnames: exactly `["title", "text"]` (order flexible). Any other column or missing column raises `CsvFormatError`. For each row: clean, compute dedup key, check against both DB and in-plan set; append to `importable_rows` or increment `duplicate_count`.
9. `csv_import.execute`: iterate `importable_rows`; before each row, call `cancel_flag()`; if true, set `cancelled=True` and break. Call `create_poem` inside a try/except; on exception record `error` and break. After the loop, return `ImportResult`.
10. `cli.py` parses `argv`, builds a `Config`, opens a DB connection, constructs an `EmbeddingService`, runs `plan` then `execute`, prints counts, handles `CsvFormatError` / `DuplicatePoemError` gracefully.
11. `__main__.py` routes based on `sys.argv[1]`: `import-csv` dispatches to `cli.import_csv`; otherwise run the Flask server.
12. Tests use a real session-scoped `EmbeddingService` to avoid repeated model loads and a fresh SQLite DB per test.

## 18. Failure Handling & Rollback
- Failure scenarios:
  - Model not found on disk → `EmbeddingService.__init__` raises with an explicit "model X not found at Y — pre-download required" message; CLI exits non-zero.
  - CSV parse failure → `plan` raises `CsvFormatError`; no writes occur.
  - Row-level write failure during `execute` → stops after the failing row; prior rows remain committed per design doc §6.3.2.
  - Cancellation flag set → stops at next boundary; prior rows remain committed.
- Partial failure handling: `execute` commits each row in its own transaction so prior success survives a mid-file failure.
- Rollback strategy: No rollback of already-imported rows (explicit per design doc §6.3.2). On a row failure the `ImportResult.error` is surfaced to the caller.
- Idempotency considerations: Duplicate detection by `cleaned_text` makes re-runs of the same CSV a no-op.

---

## 19. Risks and Edge Cases
- Risk: Model download attempted at runtime → mitigated by forcing `SentenceTransformer` to an explicit local path and verifying model files exist before constructing the model.
- Risk: Large CSV rows exceed Python's default `csv` field size limit → raise `csv.field_size_limit` to a safer upper bound at module load.
- Edge case: Empty title → allowed; embedding input becomes just the text.
- Edge case: Leading BOM in CSV → handled via `utf-8-sig`.
- Edge case: Trailing blank line in CSV → skipped silently.
- Edge case: CSV with duplicate rows within the same file → only the first copy is importable; subsequent copies count as duplicates.

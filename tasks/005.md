# Task 005: Admin Authentication Gate and Sorted Poem List

## 0. Complexity Assessment
- Level: Medium
- Justification:
  - Technical complexity: Session-based auth, a protected blueprint, templates, and a repository sort API.
  - Integration complexity: Introduces a new blueprint and interacts with the existing repository.
  - Risk level: Medium — authentication code must not leak the password or accept defaults unsafely.
  - Estimated effort: ~400 changed lines.
- Complexity rubric alignment:
  - Medium: multiple components and/or persistence involved ← fits here (auth + repository + templates).
- Recommended execution mode:
  - Medium → standard implementation with tests.

## 1. Goal
Add a password-protected admin area with login, logout, and a dashboard that lists every poem using six selectable sort orders so an administrator can review the corpus before adding CRUD and import features.

---

## 2. Why this task exists now
Tasks 006, 007, and 008 all require a protected admin surface to plug their actions into. Delivering the auth gate and the sorted list first produces an observable milestone and unblocks the remaining admin work.

## 6. Observable Outcome
- Visiting `/admin` while unauthenticated redirects to `/admin/login`.
- Submitting the wrong password re-renders the login form with a visible `Incorrect password` message.
- Submitting the correct password redirects to `/admin` and shows a poem list with id, title (or `Untitled`), created-at, and updated-at columns.
- Selecting a different sort option from the dropdown and submitting re-orders the list.
- Clicking `Logout` ends the session and returns to the login page with a `Logged out` message.
- The public search page and `/search` continue to work without authentication.

## 11. Public Functions / Methods
### auth.is_authenticated
- Purpose: Return whether the current Flask session has `admin_authenticated = True`.
- Parameters: none.
- Returns: `bool`.

### auth.login
- Purpose: Set the session flag and `session.permanent = False` so it expires on browser close.
- Parameters: none.
- Returns: none.

### auth.logout
- Purpose: Clear the session flag.
- Parameters: none.
- Returns: none.

### auth.verify_password
- Purpose: Compare the submitted password to the configured password using `hmac.compare_digest` to avoid timing attacks.
- Parameters: `submitted: str`.
- Returns: `bool`.

### auth.login_required
- Purpose: Decorator that redirects unauthenticated requests to `/admin/login`.
- Parameters: `view` callable.
- Returns: wrapped view.

### admin.login_view (GET, POST)
- Purpose: Render the login form or process a submitted password.
- Returns: rendered `login.html` or redirect to `/admin`.

### admin.logout_view
- Purpose: Clear session and redirect to the login page.

### admin.dashboard
- Purpose: Render the poem list sorted by the requested order. Query parameter `sort` controls order; default is `title_asc`.

### repository.list_poems
- Purpose: Return all poems in the requested order.
- Parameters: `conn`, `order_by` string (one of six whitelisted keys).
- Returns: list of `sqlite3.Row`.
- Errors: `ValueError` on an unknown sort key.

## 14. State Transitions
- Before: no admin area exists; `/admin` returns 404.
- After: `/admin` is a protected page; authenticated admins see the poem list with sort controls.
- Invariants:
  - Public routes still require no authentication.
  - Session cookie uses the Flask `SECRET_KEY`.
  - The configured password is never sent to the client or logged to stdout.
  - `POEM_ADMIN_PASSWORD` must be set at startup; an unset/empty value aborts app creation.

## 15. Detailed Implementation Notes
1. In `config.py`, `Config.from_environment` keeps reading `POEM_ADMIN_PASSWORD`. In `create_app`, raise `RuntimeError("POEM_ADMIN_PASSWORD must be set")` if the value is empty.
2. `auth.py` stores the session key as `"admin_authenticated"`. The decorator inspects the Flask `session`.
3. `auth.verify_password` uses `hmac.compare_digest(submitted.encode("utf-8"), configured.encode("utf-8"))`.
4. Admin blueprint prefix is `/admin`. `url_prefix="/admin"`.
5. Login view: on GET render the form; on POST verify the password; on success call `auth.login`, flash `Welcome`, and redirect to the dashboard; on failure flash `Incorrect password` and re-render the form.
6. Logout view: POST only (to prevent CSRF-via-GET); clears the session and redirects to the login page with a flash.
7. Dashboard view: read `request.args.get("sort", "title_asc")`, validate against the whitelist, fall back to the default if unknown, call `repository.list_poems(conn, order_by=sort)`.
8. Templates use `Untitled` for empty titles in the list.
9. `_base.html` provides the admin nav bar shared across login and dashboard; login page hides the Logout button via a block override.

1. `create_app` raises a clear error when `POEM_ADMIN_PASSWORD` is empty or unset.

6. `POST /admin/logout` clears the session and redirects to `/admin/login`.
7. The public search page and `/search` remain unauthenticated and functional.
8. All tests pass.

## 18. Failure Handling & Rollback
- This task adds authentication state to the session; there are no persistent data mutations.
- Failure scenarios:
  - Session cookie tampering → Flask rejects invalidly-signed cookies.
  - Missing password env var at startup → app creation raises.
- Idempotency: login and logout are idempotent at the session level.

---

## 19. Risks and Edge Cases
- Risk: Hardcoding the password in source — explicitly forbidden. Mitigated by reading from the environment and by a test asserting the empty-password failure path.
- Risk: Timing attacks on password comparison — mitigated via `hmac.compare_digest`.
- Edge case: Session cookie shared across tabs — acceptable per design doc §6.2.1.
- Edge case: Password containing Unicode — encoded as UTF-8 before comparison.

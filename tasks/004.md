# Task 004: Poem Modal Viewer with Copy Button

## 0. Complexity Assessment
- Level: Low

  - Integration complexity: Consumes the result list produced by Task 003 via `data-poem-id` attributes already present.
  - Risk level: Very low — additive, read-only, no persistence.

  - Low: single component (modal widget) with no persistence and minimal side effects ← fits here.
- Recommended execution mode:

## 1. Goal
Allow a user to click any search result to open the full poem in an overlay modal that preserves line breaks, displays the title (or `Untitled`), and exposes a Copy button that copies the full poem text to the clipboard.

---

## 2. Why this task exists now
Task 003 delivers the ranked result list but the user still cannot read a poem. This task completes the public user experience — clicking a result and reading the full text — which is a V1 acceptance requirement.

---

## 3. Prerequisites
- Task 001: scaffold and templates.
- Task 002: repository with stored poems.
- Task 003: search route and result list with `data-poem-id` attributes.

---

## 4. Scope
- New `GET /poems/<int:id>` public route returning a JSON object `{id, title, text}`.
- `_poem_modal.html` partial containing the empty modal markup included at the bottom of `search.html`.
- `static/js/search.js` with:
  - a click handler on result items that fetches `/poems/<id>` and populates the modal,
  - an open/close controller (click to open, Escape key or close button to close, click outside the content to close),
  - a Copy button that writes the full poem text to the clipboard using the async clipboard API with a fallback using a hidden `<textarea>`.
- CSS for the modal overlay, the poem `<pre>` region, and the Copy button feedback state.
- `Untitled` rendered when title is empty in both the result list (already in Task 003) and the modal header.

---

## 5. Out of Scope
- Editing, admin actions, authentication.
- Highlighting matched lines (explicitly out of V1 per design doc §6.1.5.4).
- Displaying similarity scores or explanations in the modal.
- Keyboard navigation between results (explicitly out of V1 per design doc §2.2.8).

---

## 6. Observable Outcome
- After performing a search, clicking any result row opens an overlay modal with the full poem text in a preformatted block that preserves line breaks.
- The modal header shows the poem title or `Untitled`.
- The Copy button in the modal copies the poem body to the clipboard, confirmed by a transient `Copied` label on the button.
- Pressing Escape, clicking the close button, or clicking the dimmed backdrop closes the modal.
- The background search results remain visible and unchanged when the modal closes.

## 11. Public Functions / Methods
### public.get_poem
- Purpose: Return the full text of a single poem by id as JSON for the modal.
- Parameters: path `int` id.
- Returns: JSON `{id, title, text}` with `Content-Type: application/json`. 404 if the id does not exist.
- Side effects: reads the DB.
- Errors: 404 when the poem is missing.

### search.js exported behavior (module-level)
- `initModal()`: attaches event listeners on document load — one delegated click listener on the results region, one Escape keydown listener on document.
- `openModal(id)`: fetches `/poems/<id>`, populates the modal header and body, removes the `hidden` class.
- `closeModal()`: re-hides the modal and clears its content.
- `copyPoem()`: copies the current poem text to the clipboard and toggles the `Copied` state on the button for 1500 ms.

## 13. UI / UX Changes
- Screens:
  - Modal overlay centered over the results page, dimmed backdrop.
  - Modal header shows the poem title or `Untitled`.
  - Modal body uses `white-space: pre-wrap` so line breaks are preserved visually.
  - Footer row with `Copy` and `Close` buttons.
- Interactions:
  - Click a result → modal opens.
  - Click the backdrop, the close button, or press Escape → modal closes.
  - Click Copy → text copied; button text changes to `Copied` for ~1.5 s.
- States and transitions:
  - Hidden (default), Loading (brief, while fetching), Visible.
  - Copy button has Idle and Copied states.
- Error states:
  - Fetch failure → the modal displays an inline error message `Could not load poem.` and a Retry button is not required (user can close and click again).

---

## 14. State Transitions
- Before: Clicking a result has no effect.
- After: Clicking a result opens a modal with the full poem text; the underlying results state is unchanged.
- Invariants:
  - The modal is hidden unless a fetch has just succeeded.
  - Closing the modal restores exactly the prior page state (no DOM mutation besides toggling classes on the modal container).
  - The Copy button never reveals the poem text in any other DOM location than the modal body.

## 15. Detailed Implementation Notes
1. Add the route in `public.py`: query the DB with `repository.get_poem(conn, id)`; if `None`, `abort(404)`; else return `jsonify({"id": row["id"], "title": row["title"], "text": row["text"]})`.
2. Render `_poem_modal.html` with an outer `<div class="modal" hidden>`, an inner card containing `<h2 data-modal-title>`, `<pre data-modal-body>`, and the footer button row.
3. `search.js` is plain ES modules-free JS loaded as a classic script. Use `document.addEventListener("DOMContentLoaded", initModal)`. Keep to vanilla DOM APIs so no build step is needed.
4. Delegated click handler on the results section inspects `event.target.closest("[data-poem-id]")`; if present, read the id and call `openModal(id)`.
5. `openModal` disables the results region's `aria-hidden` and traps focus minimally (just focus the Close button on open).
6. Empty title: if `title.trim() === ""` render `Untitled`.
7. `copyPoem` uses `navigator.clipboard.writeText(text)` when available. Fallback: create a hidden textarea, select, `document.execCommand("copy")`, remove. After success, set button text to `Copied`, then restore to `Copy` after 1500 ms.
8. Escape key handler closes the modal only when the modal is visible.
9. CSS: `.modal[hidden] { display: none }`, `.modal` positions the overlay absolutely over the viewport, `.modal__card` centers a max-width card; `.modal__body` has `white-space: pre-wrap; font-family: inherit;`.

## 16. Acceptance Criteria
1. `GET /poems/<id>` returns JSON with the full title and text for an existing poem; 404 otherwise.
2. Clicking a result opens the modal and populates it with the correct poem.
3. Line breaks from the stored text appear as line breaks in the modal.
4. The Copy button copies the full poem text to the clipboard.
5. Pressing Escape closes the modal; clicking the backdrop or Close button also closes it.
6. Search results remain unchanged when the modal closes.
7. Empty title renders as `Untitled` in the modal header.
8. All tests pass against a real SQLite DB.

## 18. Failure Handling & Rollback
- This task introduces no mutations; rollback does not apply.
- Failure scenarios:
  - Fetch returns 404 → JS displays `Could not load poem.` inside the modal body; user can close and retry.
  - Clipboard API unavailable → fallback textarea method runs; if that also fails the button text changes to `Copy failed` for 1500 ms.

---

## 19. Risks and Edge Cases
- Edge case: Very long poems exceed viewport height → modal card has `max-height: 80vh` and `overflow-y: auto` on the body.
- Edge case: Poems with Unicode → JSON handles Unicode natively; template uses `{{ ... | e }}` only where appropriate.
- Risk: Clipboard API blocked on non-HTTPS origins → localhost is allowed by all major browsers; documented in the README.
- Edge case: Rapid clicks on multiple results → the second fetch replaces the first; the `openModal` function aborts any in-flight fetch using `AbortController`.
